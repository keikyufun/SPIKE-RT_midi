# Web変換ツール ガイド

ブラウザベースのSPIKE-RT MIDI変換ツールの詳細なガイドです。

## 概要

Web版変換ツール(convert.html)は、MIDI音声をSPIKE RTプロジェクト用のC言語コードに変換するブラウザアプリケーションです。

### 機能

- **MIDIファイルアップロード**: ファイルシステムからMIDI選択
- **トラック選択**: マルチトラックMIDIの個別トラック処理
- **パート分割**: 和音を複数パートに自動分割
- **テンポ抽出・カスタマイズ**: MIDI埋め込みテンポを自動読取・手動指定可能
- **オーディオプレビュー**: Web Audio APIで再生確認
- **C言語コード生成**: SPIKE RT対応コード自動生成
- **複数出力形式**: 単一パート/全パート変換
- **ダークグリーンテーマ**: ユーザーフレンドリーなUIデザイン

## 使用方法

### 基本的な使い方

#### 1. ファイル読み込み

```
[Load MIDI File]ボタンをクリック
  → ファイルダイアログが表示
  → MIDIファイルを選択
  → ファイル情報が表示
```

表示される情報：
- **Tempo (BPM)**: MIDI内に設定されたテンポ
- **Tracks**: トラック数
- **Current Track**: 現在選択中のトラック
- **Parts**: 現在のトラック内のパート数
- **Notes**: 各パートのノート数

#### 2. テンポ指定（オプション）

```
「テンポ(BPM)」フィールドに値を入力（1-500）
  → 空白の場合: MIDIから自動検出
  → 数値入力: その値でコード生成
```

「プレビュー」ボタンをクリック時に、テンポ情報が表示されます：
- `テンポ: 120 BPM` （自動検出）
- `テンポ: 120 BPM (指定テンポ: 150 BPM)` （カスタム指定時）

#### 2.5. トラック選択

```
ドロップダウン「Track」から選択
  → プレビュー情報が自動更新
  → パート数とノート情報が変更
```

#### 3. データプレビュー

```
[Preview]ボタンをクリック
  → ノート情報が表示
  → 各パートの詳細を確認
```

表示内容：
- パートごとのノート一覧
- 各ノートの開始時刻(ms)
- ノート長さ(ms)
- 周波数(Hz)

#### 4. オーディオテスト

```
[Play]ボタンをクリック
  → Web Audio APIで再生
  → ブラウザスピーカーから音声出力
```

#### 5. コード生成

```
[Convert]ボタンをクリック
  → C言語コード自動生成
  → テキストボックスに表示
```

#### 6. コード確認・コピー

```
テキストボックスのコードを確認
  → Ctrl+A で全選択
  → Ctrl+C でコピー
  → テキストエディタに貼り付け
```

## 機能詳細

### トラック処理

#### 単一トラック変換

```
1. トラックをドロップダウンから選択
2. [Convert]をクリック
3. 1つのトラックのコードが生成
```

#### 全トラック変換

Web版では単一トラック処理に対応しています。
全トラック一括処理はPython版またはC言語版をご利用ください。

```bash
# Python版で全トラック変換
python3 midi_to_spike.py song.mid --all --fast

# C言語版で全トラック変換
./main song.mid --all --fast
```

### 和音分割(パート自動生成)

同一トラック内で和音(複数ノートの同時刻)が検出されると、自動的に複数パートに分割します。

**例:**
```
元のトラック: C4 E4 G4 (同時刻)
     |
     v
 分割後:
 Part 0: C4
 Part 1: E4
 Part 2: G4
```

各パートは別々のコード行で生成されます。

### テンポ処理

#### テンポ自動抽出

MIDI内の「Set Tempo」メタイベント(FF 51 03)から自動抽出します。

```
公式: BPM = 60,000,000 / マイクロ秒単位のビート長
```

#### テンポが見つからない場合

デフォルト: 120 BPM

#### 生成コード内のテンポ

```c
const int tempo = 170;  // BPM
const int beat_ms = 60000 / tempo;  // 1ビート = 353ms
```

## 出力コード形式

### 単一パートの場合

```c
const int tempo = 170;
const int beat_ms = 353;
const int note_data[][2] = {
    {440, 375},
    {494, 375},
    {523, 750},
    {0, 0}
};
```

### 複数パートの場合

各パートが別々に生成されます：

```c
/* Track 0 Part 0 */
const int tempo = 170;
const int beat_ms = 353;
const int note_data[][2] = {
    {440, 375},
    {494, 375},
    {0, 0}
};

/* Track 0 Part 1 */
const int tempo = 170;
const int beat_ms = 353;
const int note_data[][2] = {
    {523, 750},
    {587, 375},
    {0, 0}
};
```

### コード要素の説明

| 要素 | 説明 |
|------|------|
| `tempo` | MIDI内のテンポ(BPM) |
| `beat_ms` | 1ビートの長さ(ミリ秒) |
| `note_data` | ノート情報 |
| `note_data[i][0]` | 周波数(Hz) |
| `note_data[i][1]` | ノート長さ(ms) |
| `{0, 0}` | リスト終端 |

## MIDI ノート情報

### ノートナンバーから周波数への変換

```
周波数 = 440 * 2^((ノートナンバー - 69) / 12)
```

### よく使う周波数

| ノート | 周波数(Hz) |
|--------|-----------|
| C4 | 262 |
| D4 | 294 |
| E4 | 330 |
| F4 | 349 |
| G4 | 392 |
| A4 | 440 |
| B4 | 494 |
| C5 | 523 |

## トラブルシューティング

### MIDIファイル関連

| 問題 | 原因 | 解決方法 |
|------|------|---------|
| ファイルが開けない | ファイル形式が MIDI でない | .mid または .midi 拡張子確認 |
| 情報が表示されない | MIDI ヘッダが無効 | 別の MIDI ファイルで試す |
| テンポが 120 | テンポ情報が未設定 | MIDI ファイルの確認 |

### 再生関連

| 問題 | 原因 | 解決方法 |
|------|------|---------|
| 無音 | ブラウザの音量 | ブラウザボリューム/システムボリューム確認 |
| 音が変 | ノート情報が不正 | MIDI ファイルの検証 |
| 途中で止まる | メモリ不足 | ブラウザの再起動 |

### コード生成関連

| 問題 | 原因 | 解決方法 |
|------|------|---------|
| コードが空 | トラックにノートなし | 別のトラック選択 |
| テンポが反映されない | MIDI テンポ未検出 | デフォルト値は 120 BPM |
| ノート数が多い | メモリ制限 | ファイルを分割 |

## 使用例

### 例1: シンプルなメロディ

```
1. sample.mid をアップロード
2. Track 0 を選択
3. [Play] で再生確認
4. [Convert] でコード生成
5. コードをコピー
```

### 例2: 和音含むファイル

```
1. harmony.mid をアップロード
2. Track 0 を選択
3. Parts が複数表示される(和音分割済み)
4. [Preview] で各パート確認
5. [Convert] でコード生成(各パート別)
```

### 例3: 高速なメロディ

```
1. fast_song.mid をアップロード
2. [Preview] でノート情報確認
3. ノートが多い場合: [Convert] でコード生成
4. Python版を使用: python3 midi_to_spike.py fast_song.mid
```
